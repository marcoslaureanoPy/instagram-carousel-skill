# Generación de Contenido para Carruseles

## Estructura de un carrusel completo

Un carrusel tiene entre **5 y 8 slides**. La estructura estándar es:

| Slide | Nombre | Función |
|-------|--------|---------|
| 1 | Hook | Captar atención, generar curiosidad |
| 2 | Problema/Contexto | Conectar con el dolor o situación del lector |
| 3-6 | Desarrollo | Tips, pasos, puntos principales (numerados) |
| N-1 | Resumen/Bonus | Cierre del contenido, tip extra |
| N (último) | CTA | Call to action + marca |

## Reglas de copywriting por tono

### Tono A — Profesional y formal
- Usar "usted" o tercera persona
- Frases completas, sin jerga
- Datos y fuentes cuando aplique
- Ejemplo hook: *"3 errores estratégicos que frenan el crecimiento de su negocio"*

### Tono B — Cercano y amigable
- Usar "tú" o "vos" según región
- Hablar como a un amigo
- Emojis ocasionales en el copy (no en el SVG)
- Ejemplo hook: *"¿Te ha pasado esto con tus clientes? Sigue leyendo 👀"*

### Tono C — Inspirador y motivacional
- Frases poderosas y aspiracionales
- Verbos de acción: transforma, logra, conquista
- Ejemplo hook: *"Lo que separa a los exitosos del resto es este hábito"*

### Tono D — Educativo y didáctico
- Paso a paso claro
- Lenguaje accesible
- Analogías para conceptos complejos
- Ejemplo hook: *"Te explico en 5 slides cómo funciona [tema]"*

### Tono E — Divertido y desenfadado
- Referencias culturales si aplica
- Humor suave
- Contraste sorpresivo
- Ejemplo hook: *"Nadie te dijo esto sobre [tema] y ya era hora"*

## Reglas de extensión de texto por slide

| Slide | Título (caracteres) | Subtexto (palabras) |
|-------|---------------------|----------------------|
| Hook | 40-60 | 10-20 palabras opcionales |
| Problema | 40-60 | 15-25 palabras |
| Desarrollo (cada uno) | 30-50 | 15-30 palabras |
| Resumen | 30-50 | 10-20 palabras |
| CTA | 20-40 | 10-15 palabras |

**Regla de oro:** Si el texto no cabe en una tarjeta de 1080x1080, es demasiado largo. Corta.

## Tipos de contenido y sus estructuras

### Tips y consejos (tipo A)
```
Slide 1: "[Número] tips para [lograr X] si eres [audiencia]"
Slide 2: El problema que resuelven estos tips
Slide 3-N: "Tip #1: [Nombre]" + explicación breve
Último: "Guarda esto para cuando lo necesites" + CTA
```

### Errores comunes (tipo B)
```
Slide 1: "Estás cometiendo estos errores en [área] (sin darte cuenta)"
Slide 2: Por qué estos errores te cuestan caro
Slide 3-N: "Error #1: [nombre]" + cómo evitarlo
Último: "¿Cuál de estos cometías?" + CTA
```

### Tutorial paso a paso (tipo C)
```
Slide 1: "Cómo [lograr resultado] en [tiempo/pasos]"
Slide 2: Lo que necesitas antes de empezar
Slide 3-N: "Paso #1: [acción]" + detalle
Último: "Resultado esperado" + CTA
```

### Mitos vs Realidad (tipo D)
```
Slide 1: "Todo lo que te dijeron sobre [tema] está MAL"
Slide 2: De dónde vienen estos mitos
Slide 3-N: "Mito: [afirmación]" / "Realidad: [corrección]"
Último: La verdad que necesitas saber + CTA
```

### Datos y estadísticas (tipo G)
```
Slide 1: "El dato sobre [tema] que nadie te dice"
Slide 2: Contexto del dato principal
Slide 3-N: "[Dato]%" / "[Estadística]" + qué significa
Último: Qué hacer con esta información + CTA
```

## Output esperado de esta fase

Antes de generar los SVGs, presenta el contenido en texto plano así:

```
📋 CONTENIDO DEL CARRUSEL: "[Tema]"

SLIDE 1 (Hook)
Título: [texto]
Subtexto: [texto o "ninguno"]

SLIDE 2 (Problema)
Título: [texto]
Subtexto: [texto]

SLIDE 3 (Punto 1)
Número: 01
Título: [texto]
Subtexto: [texto]

[... continuar para cada slide ...]

SLIDE N (CTA)
Título: [texto]
Subtexto: [texto]
Handle: @[usuario]
```

Luego pregunta:
```
¿Apruebas este contenido o quieres ajustar algo antes de generar los diseños?
(sí / [lo que quieras cambiar])
```

Solo continúa a los SVGs si el usuario aprueba.
